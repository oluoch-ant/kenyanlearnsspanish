# Setup — git deploys + cross-device sync

Everything in this folder is ready. What's left are the four things that need your hands:
a repo, a link, a secret, and a passphrase. About 15 minutes, once.

---

## What changes

**Before:** edit the file → copy it over `pwa/index.html` → remember to bump `sw.js` → drag the
`pwa` folder onto Netlify. Four steps, no version history, no undo.

**After:** edit the file → `git push`. That's it. The build does the copy and stamps the
service-worker version with the commit sha, so your phone always picks up the new build.

And your progress syncs between the phone and the desktop.

---

## 1. Put it in git (5 min)

In **Git Bash**, from this folder:

```sh
cd "/c/Users/oluoc/Spanish Teacher"
git init
git add -A
git commit -m "Spanish Teacher: Core 8 mastery, new navigation, cross-device sync"
```

Then make a repo on GitHub and push. If you have the `gh` CLI:

```sh
gh repo create kenyanlearnsspanish --private --source=. --push
```

Otherwise create an empty repo at github.com and follow the "push an existing repository"
lines it gives you.

**The repo can be public or private — there are no secrets in the code.** The sync passphrase
lives in Netlify, not in the files.

## 2. Link the Netlify site to the repo (3 min)

On https://app.netlify.com → your **kenyanlearnsspanish** project →
**Project configuration** → **Build & deploy** → **Link repository**.

Pick the repo. Netlify will read `netlify.toml` and fill these in itself — just check they match:

| Setting | Value |
|---|---|
| Build command | `node scripts/build.mjs` |
| Publish directory | `pwa` |
| Functions directory | `netlify/functions` |

**Your site keeps the same URL** (`kenyanlearnsspanish.netlify.app`), so the PWA already
installed on your phone keeps working and does **not** need reinstalling.

## 3. Set the sync secret (2 min)

Same project → **Project configuration** → **Environment variables** → **Add a variable**:

- **Key:** `SYNC_SECRET`
- **Value:** a long random string. Make one up, or run `openssl rand -hex 24` in Git Bash.

Keep a copy — you'll type it into each device once, in step 4.

Then **Deploys → Trigger deploy → Deploy site**.

## 4. Turn sync on, on each device (2 min)

Open https://kenyanlearnsspanish.netlify.app on the **desktop** (in the browser — not the file
off your disk any more; that's the old way and it has its own separate, stranded progress).
Click **Install** in the address bar to keep it as a desktop app.

Then: **Progress tab → 🔑 Sync passphrase →** paste the `SYNC_SECRET` → it syncs immediately.

Do the same on the **phone**. That's it. From then on it syncs on open, when you come back
online, and a few seconds after you stop drilling.

---

## How the sync behaves

- **Local first.** Everything saves to the device instantly. Nothing ever waits on the network.
- **Offline is fine.** Drill on the plane; it syncs when you're back.
- **Both devices before a sync is fine.** The server *merges* — it doesn't overwrite. Reps done
  on the phone and reps done on the desktop both survive. A verb you got to 2/3 days on one
  device and 1/3 on the other comes out at 3/3 and masters.
- **It won't double-count.** Two devices drilling the same thing on the same day is still one
  rep for that day — the one-rep-per-day rule holds across devices.
- **Streak is never punished.** If the two devices disagree, the higher streak wins.
- **Export/import still works** as a manual backup, and as your escape hatch if you ever want
  off Netlify.

## What it costs you in trust

The passphrase sits in `localStorage` on each device, like your Claude API key. The difference
is what it unlocks: **anyone who steals it can read and scramble your Spanish verb reps, and
nothing else.** It is not tied to your GitHub, your Netlify account, or anything that costs
money. That was the whole reason for putting the secret in a Netlify environment variable
instead of putting a GitHub token in the browser.

## Deploying from now on

```sh
git add -A
git commit -m "what changed"
git push
```

Netlify builds and deploys. Watch it under **Deploys**. If a build fails, the live site stays
on the last good deploy — your phone never sees a broken build.

You can still drag-and-drop the `pwa` folder in an emergency, but the folder's `index.html` is
generated by the build now, so run `node scripts/build.mjs` first if you do.
